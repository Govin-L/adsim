# AdSim

> Simulate before you spend. 投放之前，先模拟。

基于 AI Agent 的营销方案评估系统。当前版本重点支持结构化方案解析、基于素材描述的 placement 级行为模拟，以及从 Agent 反应中提炼洞察。像 CPA 这类业务指标目前仍是结合行业基准的估算值，而不是纯涌现结果。

## 工作原理

AdSim 通过 LLM 驱动的用户 Agent 群体模拟，帮助你在投放前了解方案可能会遇到什么问题、哪类人群会如何反应。它的核心价值不是预测准确的 CPA 数字，而是提供结构化的洞察：

1. **生成用户 Agent** — 创建代表不同平台用户的人群样本，每个 Agent 有独立的人口特征、兴趣偏好和行为模式
2. **模拟行为决策** — 基于素材描述和市场环境，让每个 Agent 分阶段判断：会不会注意到？会不会点击？会不会购买？每次判断都输出结构化的推理标签
3. **聚合结构化洞察** — 从 Agent 群体的决策推理中提炼流失原因聚类、人群画像差异、关键阻塞因素和可操作建议
4. **解释"为什么"** — 可以采访任何 Agent 了解其决策原因和阻塞因素
5. **业务指标估算** — CTR/CVR 等指标来自模拟统计，CPA 等业务指标结合预算和行业基准进行估算，标注为参考值

## 当前能力

- **流失原因聚类** — 从 Agent 推理中自动聚合 5-8 个关键流失原因，附带占比和代表性反馈
- **人群画像洞察** — 按年龄、性别、收入、价格敏感度等维度对比转化差异，自动标注最高/最低人群
- **placement 级漏斗** — 查看每个投放位的注意→点击→转化漏斗，含阶段阻塞因素和优化建议
- **Agent 采访** — 深入了解任意 Agent 的决策原因和阻塞因素
- **市场环境输入** — 支持竞品、品牌知名度、投放目标类型等上下文
- **业务指标参考** — CTR/CVR 等模拟统计 + 基于行业基准估算的 CPA，标注为参考值

## 正在建设

- **多 placement 整案编排** — 将多个 placement 作为一整套方案协同评估
- **优化建议** — 基于同一批样本对候选方案做反事实验证
- **投后校准** — 回填实际结果，逐步修正业务指标估算

## 技术栈

- **后端**: Kotlin + Spring Boot 3 + Langchain4j
- **前端**: React + Vite + Tailwind CSS + shadcn/ui
- **数据库**: MongoDB + Redis
- **部署**: Docker Compose

## 哲学反思：多 Agent 模拟的「涌现」幻觉

> **添加于 2026-05-13。** 这是一个开放问题，欢迎讨论。

自项目启动以来，我们反复问自己一个问题：这种 Agent 模拟真的能「涌现」出超越提示词设计的智慧吗？

### 问题：曲线拟合 vs 真正涌现

目前多数多 Agent 模拟，包括 AdSim 在内，本质上和量化交易的回测没有区别：

| 量化交易 | 多 Agent 模拟 |
|---------|-------------|
| 调参数拟合历史 K 线 | 调 prompt / Agent 数量 / 决策流程拟合预期效果 |
| 回测漂亮，实盘崩 | demo 惊艳，换场景崩 |
| 过拟合 | 针对特定方案调出来的「智能」 |

真正的涌现应该是系统层面的相变（phase transition），而不是开发者手调参数让输出更好看。如果涌现是能被「设计」出来的，那它就不叫涌现了。

### 根源：同质化的 Agent 群体

大多数多 Agent 框架的所有 Agent 共享同一个底层 LLM。AdSim 也不例外。这意味着：

- 共享同样的知识边界
- 共享同样的推理缺陷
- 共享同样的认知偏差

一群完全同质的个体凑在一起，不可能涌现出超越个体的智慧。这本质上是多数投票（majority voting）的豪华版——统计意义上更稳定了，但没产生新的东西。

### 对 AdSim 的启示

承认这个局限，不是要否定 Agent 模拟的价值。AdSim 的价值在于**结构化洞察**，而非涌现预测：

1. **流失原因聚类**是有价值的——它帮人类决策者看到了群体的共性反应
2. **人群画像差异**是有价值的——它展示了不同人群在同一素材下的行为差异
3. **Agent 采访**是有价值的——它把 LLM 的推理能力锚定在一个具体的决策场景下

这些能力不需要「涌现」，它们来自 LLM 本身就有的推理能力和结构化输出。

### 真正的突破口：异质性

如果未来要突破当前局限，我们认为方向是**引入 Agent 异质性**：

- **模型层**：不同 Agent 使用不同模型（一个擅长分析、一个偏向直觉、一个更谨慎）
- **知识层**：每个 Agent 拥有独立的记忆和知识库，模拟真实用户的个性化认知
- **人格多样性**：不只是年龄性别这些人口标签，而是真正的决策风格差异
- **不一致的行为模式**：相同输入下不同 Agent 给出不同反应，而不是统一被同一 LLM 驱动

### 结论

**智能涌现的前提，是异质性。**

如果所有 Agent 都是同一个人的不同分身，它不会比一个人更强。AdSim 当前的价值不依赖于涌现，而我们有意识地选择了在这个认知框架下做产品。未来如果突破了 Agent 异质性的瓶颈，涌现才有可能成为真正的 feature。
