# AdSim

> Simulate before you spend. 投放之前，先模拟。

基于 AI Agent 的营销方案评估系统。当前版本重点支持结构化方案解析、基于素材描述的 placement 级行为模拟，以及从 Agent 反应中提炼洞察。像 CPA 这类业务指标目前仍是结合行业基准的估算值，而不是纯涌现结果。

## 工作原理

AdSim 通过 LLM 驱动的用户 Agent 群体模拟，帮助你在投放前了解方案可能会遇到什么问题、哪类人群会如何反应。它的核心价值不是预测准确的 CPA 数字，而是提供结构化的洞察：

1. **生成用户 Agent** — 创建代表不同平台用户的人群样本，每个 Agent 有独立的人口特征、兴趣偏好和行为模式
2. **模拟行为决策** — 基于素材描述和市场环境，让每个 Agent 分阶段判断：会不会注意到？会不会点击？会不会购买？每次判断都输出结构化的推理标签
3. **聚合结构化洞察** — 从 Agent 群体的决策推理中提炼流失原因聚类、人群画像差异、关键阻塞因素和可操作建议
4. **解释"为什么"** — 可以采访任何 Agent 了解其决策原因和阻塞因素
5. **业务指标估算** — CTR/CVR 等指标来自模拟统计，CPA 等业务指标结合预算和行业基准进行估算，标注为参考值

## 当前能力

- **流失原因聚类** — 从 Agent 推理中自动聚合 5-8 个关键流失原因，附带占比和代表性反馈
- **人群画像洞察** — 按年龄、性别、收入、价格敏感度等维度对比转化差异，自动标注最高/最低人群
- **placement 级漏斗** — 查看每个投放位的注意→点击→转化漏斗，含阶段阻塞因素和优化建议
- **Agent 采访** — 深入了解任意 Agent 的决策原因和阻塞因素
- **市场环境输入** — 支持竞品、品牌知名度、投放目标类型等上下文
- **业务指标参考** — CTR/CVR 等模拟统计 + 基于行业基准估算的 CPA，标注为参考值

## 正在建设

- **多 placement 整案编排** — 将多个 placement 作为一整套方案协同评估
- **优化建议** — 基于同一批样本对候选方案做反事实验证
- **投后校准** — 回填实际结果，逐步修正业务指标估算

## 技术栈

- **后端**: Kotlin + Spring Boot 3 + Langchain4j
- **前端**: React + Vite + Tailwind CSS + shadcn/ui
- **数据库**: MongoDB + Redis
- **部署**: Docker Compose

## 快速开始

### 环境要求

- JDK 21+
- Node.js 20+
- Docker & Docker Compose
- MongoDB & Redis（或使用 Docker Compose）

### 后端

```bash
cd server
./gradlew bootRun
```

### 前端

```bash
cd web
npm install
npm run dev
```

### Docker Compose

```bash
docker compose up -d
```
